BUG #1: Dashboard Stuck on "Loading..."
- Symptom: Page shows "Loading..." indefinitely

- Impact: User cannot access dashboard
- Fix: Changed role filter to exclude NATIONAL scope roles

BUG #2: "No Role Assigned" + "System" Scope
- Symptom: About page shows empty role and "System" scope
- Cause: Same as Bug #1 - buildAccessContext fails

- Fix: Same single-line fix resolves both issues

BUG #3: Admin Module Not Accessible
- Symptom: Admin link not visible, /admin route inaccessible
- Cause: accessContext null prevents admin permission checks
- Impact: Admins cannot access user/role management
- Fix: Same fix enables admin module access

================================================================================
ROOT CAUSE
================================================================================

File: src/contexts/AuthContext.tsx
Line: 223

BEFORE (BROKEN):
  const operationalRoles = userRoles.filter(r => r.scope_type !== null && r.scope_type !== undefined);
  // This INCLUDED all roles with non-null scope_type, including NATIONAL admin roles

AFTER (FIXED):
  const operationalRoles = userRoles.filter(r => r.scope_type && r.scope_type !== 'NATIONAL');
  // This INCLUDES only SC and CATCHMENT scoped roles (actual operational scopes)

WHY IT FAILED:
- User had 2 roles: TO (SC-scoped) + Global Admin (NATIONAL-scoped)
- Old filter treated both as "operational roles" (length = 2)
- Validation: if (operationalRoles.length > 1) throw error
- Error prevented buildAccessContext from completing
- accessContext stayed null → "Loading..." displayed forever

WHY IT'S FIXED:
- Global Admin is now correctly classified as ADMINISTRATIVE (not operational)
- operationalRoles now contains only TO (length = 1)
- Validation passes: single operational scope is valid
- buildAccessContext completes successfully
- accessContext is properly initialized
- Dashboard loads, roles displayed, admin accessible

================================================================================
ROLE CLASSIFICATION (CORRECTED)
================================================================================

OPERATIONAL ROLES (User's Working Scope):
- Scope Type: 'SC' or 'CATCHMENT'
- Purpose: Determines default working area
- Constraint: User can have ONLY ONE operational role
- Examples: TO (Transport Operator), CM (Catchment Manager)

ADMINISTRATIVE ROLES (Module Access):
- Scope Type: 'NATIONAL'
- Purpose: Enables access to admin module
- Constraint: Can be combined with operational role
- Examples: Global Admin, WSSM, Director, CEO

VALID COMBINATION:
- User with TO (SC) + Global Admin (NATIONAL) ✓ Valid
  - Works in Murombedzi SC by default
  - Can access admin module

INVALID COMBINATIONS:
- User with Global Admin only ✗ Invalid (no operational scope)
- User with TO (SC) + CM (CATCHMENT) ✗ Invalid (2 operational scopes)
